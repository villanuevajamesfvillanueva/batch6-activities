Responsive Chess Layout v2.2.1 ♜

v2.2.1 04.02.21 What's new?


  • Fixed board tile colors not in proper order.
  • Removed border radius to each tile to improve visual clarity.

    • Huge thanks to chess WIM Mikee Suede for helping me out with these!


  • Some fix to move logs for captures.
  • If scroll bar is activated, move logs auto scroll down once a new entry is added.


v2.2 04.02.21 updates:


  • Bug fix for extreme touch move. Player can now unselect preselected pieces.
  • Player movement constraints now in effect. Pieces can only be moved to their proper corresponding target tiles
  • Disabled all piece movements once a winner has been declared.


v2.1 03.02.21 updates:

  • Improved player movement constraint. Pieces can no longer land anywhere on the board
  • Disabled cursor: pointer on empty tiles and non-movable pieces.
  • Implemented a basic reset button on smaller screens.
  • Fixed chessboard file and rank labels and move logs font not appearing uniform across some devices.
  • Minor adjustment to mobile screens.
  • Reduced file size of some assets to speed up loading time.


v2.0 Known bugs features:


  • Pieces can move anywhere.
  • Pieces can capture any other piece, including the ones that belong on their side.
  • Player can control opponent's pieces.
  • Captures don't properly reflect in the move log.
  • No castling implementation yet.
  • No visual or audio cue implementations yet for checks.
  • No option yet to deselect selected piece.
  • After one side wins, if player moves a piece, the bot will still play as if the game is not over yet.

To enjoy the game, please try to play honestly against the AU (Artificial Unintelligence). Stop harassing it and insta promoting every single pawn you have into a queen. Bots have feelings too you know. Good luck, have fun!
